# 角色立繪基礎畫風

本專案 AVG 角色立繪以「簡化日本動漫立繪」為基礎風格。生成或重繪角色時，先抽取角色身份要素，再套用本文件的畫風規則；不要直接複製參考圖姿勢或服裝錯誤。

## 基礎風格

- 畫面類型：全身 AVG 立繪、單人、站姿、透明背景 PNG。
- 線條：乾淨粗黑線，輪廓清楚，髮絲與服裝內線可細一些，但避免草稿線、厚塗筆觸和雜訊。
- 上色：平面賽璐璐上色，色塊乾淨，陰影少量且邊界明確；不要水彩、油畫、厚塗、寫實光影或複雜漸層。
- 比例：日本動漫 Q 版偏修長，頭身略低於一般寫實比例，四肢簡化，手部不用過度細節。
- 表情：眼睛大且清楚，嘴鼻簡化，表情差分用眉眼和嘴型的小改動完成。
- 構圖：角色完整入鏡，腳底可見，左右留出頭髮與衣襬空間；遊戲用縮圖以 160x240 PNG 為基準。
- 背景：生成中間稿使用純色 chroma key 背景，最終交付必須是真 alpha 透明 PNG。

## 色彩規則

- 主色以高對比色塊為主，常用白、紅、黑與少量紫色眼睛高光。
- 避免複雜材質、金屬裝甲、寫實布料紋理、灰階髒污和照片感光影。
- 角色身份色應優先於模型自行添加的飾品色；不需要的貓耳、獸耳、裝甲、披風應明確排除。


## 負面約束

生成時排除：

- 厚塗、水彩、油畫、鉛筆草稿、髒污筆觸。
- 白底或假透明棋盤格作為最終輸出。

## 交付規則

- 每個角色至少保留 `neutral`、`smile`、`serious`、`surprised` 四張差分。
- 高解析中間稿與生成 metadata 放在 `art/characters/<character-id>/`。
- 遊戲用透明 PNG 放在 `project/images/`，並加入 `project/data.js -> main.images`。
- 交付前用 `validate_png_alpha.py` 驗證所有最終 PNG 有真透明像素。
